====== Планшет для рисования ======

[ @debian @install @digimen @huion @планшет ]

Отключить планшет от ноута.

На странице релизов https://github.com/DIGImend/digimend-kernel-drivers/releases в последнем релизе в разделе Assets (развернуть) скачать файл *.deb

Установить драйвера:

''sudo dpkg -i digimend-*.deb''

Установить утилиты для переназначения кнопок:

''sudo apt install xbindkeys xdotool''

Утилита xbindkeys для неподготовленных коварна: по-умолчанию она не запускается, это надо делать вручную. Но если сделать в профиле юзера файл для ее конфига — ''~/.xbindkeysrc'' — и что-то туда записать для эксперимента и ЗАБЫТЬ об этом, то после перезагрузки компьютер неожиданно начинает руководствоваться настройками этого конфига, который молча перехватывает на себя нажатие клавиш. Можно получить ноутбук, у которого курсор по экрану двигается, а по иконкам не кликает. Решение: прибить процесс через ''pkill xbindkeys'' и отредактировать (а лучше удалить) файл конфига.

Посмотреть на включенные устройства, которые распознаёт система:

''xinput list''

Подключить планшет:

* на нём загорается тусклый световой индикатор
* перо над ним работает как мышка
* клики по кнопкам на пёрышке работают как левая и правая кнопка мыши

Снова выполнить:

''xinput list''

В ответе ожидаем увидеть:

'''
⎜   ↳ HID 256c:006e Pen stylus           id=15
⎜   ↳ HID 256c:006e Pad pad              id=16
⎜   ↳ HID 256c:006e Touch Strip pad      id=17
⎜   ↳ HID 256c:006e Group Buttons pad    id=18
'''

Если виден “Pad pad” — мы в норме. 

В данном случае у планшета “id=16”, но это временно, после каждого подключения планшета id устройствам раздаётся заново и может поменяться.

===== Обновить драйвер =====

Это нужно редко, но бывает.

Раньше приходилось сперва убирать предыдущий драйвер, а сегодня можно просто установить новую версию и перезагрузить компьютер. Если перезагрузка по каким-то причинам невозможна, следует отключить планшет и выполнить

''sudo modprobe -r hid-kye hid-uclogic hid-polostar hid-viewsonic''

Подключить планшет снова.

А потом все равно придётся перезагрузить компьютер.
