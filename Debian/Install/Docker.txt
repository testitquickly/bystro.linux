====== Docker ======

[ @debian @install @docker ]

Придётся перезагрузить если не компьютер, то сессию своего юзера.

===== Установить Docker =====

''sudo apt update && sudo apt upgrade -y''

Debian 12:

''sudo apt install -y apt-transport-https ca-certificates curl gnupg software-properties-common''

Debian 13:

''sudo apt install -y apt-transport-https ca-certificates curl gnupg''

Добавить ключ Docker:

''sudo install -m 0755 -d /etc/apt/keyrings''

''curl -fsSL https://download.docker.com/linux/debian/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg''

''sudo chmod a+r /etc/apt/keyrings/docker.gpg''

Добавить репозиторий:

''echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/debian \''
''$(. /etc/os-release && echo "$VERSION_CODENAME") stable" | \''
''sudo tee /etc/apt/sources.list.d/docker.list > /dev/null''

Установить сам Docker:

''sudo apt update && sudo apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin''

==== Запустить сервис ====

''sudo systemctl start docker && sudo systemctl enable docker && sudo docker --version''

==== Добавить своего пользователя в группу docker ====

…чтобы запускать команды Docker без sudo:

''sudo usermod -aG docker $USER''

$USER — это переменная окружения, содержить текущего пользователя.

Перезагрузить компьютер.

''sudo reboot''

После перезагрузки убедиться в том, что действующий юзер вхож в группу ''docker'':

''groups''

===== Установить Docker Compose =====

Отдельный инструмент, который упрощает работу с Docker. Может управлять многоконтейнерными приложениями с помощью одного конфигурационного файла.

''sudo curl -L "https://github.com/docker/compose/releases/download/v2.22.0/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose && sudo chmod +x /usr/local/bin/docker-compose && docker-compose --version''
