====== Firefox ======

[ @debian @install @firefox ]

В базе установлен “Firefox ESR”, и это всегда заметно отстающая версия браузера. Удалить его нельзя, потому что он установлен как зависимость метапакета среды рабочего стола KDE, и при его удалении APT удалит всё, что входит в этот метапакет (половина системы). Можно его просто прихлопнуть:

1. заморозить пакет ''firefox-esr''
2. переименовать файл его вызова

''sudo apt-mark hold firefox-esr''

Обновления будут игнорировать пакет Firefox ESR. Проверить, какие еще пакеты заморожены: ''apt-mark showhold''

Убрать с глаз системы файл запуска Firefox ESR:

''sudo mv /usr/share/applications/firefox-esr.desktop /usr/share/applications/firefox-esr.desktop.disabled''

Приложение “Firefox ESR” должно исчезнуть из Меню приложений и из вызова [[Software:KRunner]]

===== Установить Firefox в /opt =====

==== Скачать архив с установкой ====

с https://www.mozilla.org/ru/firefox/new/

Обычно он раздаётся в виде файла „''firefox-x.x.xx.tar.bz2''”

==== Распаковать архив ====

Из него надо достать каталог „firefox” с файлами браузера внутри. 

==== Переместить файлы в /opt ====

Разумно держать весь «сторонний» софт в одном месте, поэтому  

''sudo mv firefox /opt''

==== Дать права своему пользователю на изменения в каталоге /opt/firefox ====

''sudo chown -R'' __''astenix''__'':sudo /opt/firefox && sudo chmod 750 /opt/firefox''

==== Сделать symlink на firefox ====

Этого достаточно для запуска Firefox из CLI.

Для установки нового Firefox только для своего пользователя:

''cd ~/.local/bin/ && ln -s /opt/firefox/firefox''

=== Сделать symbolic link на firefox — для всех пользователей ===

Это инструкция для установки нового Firefox для всех пользователей.

Проверить, что в ''/usr/bin/'' нет файла для запуска firefox:

''ls /usr/bin/firefox''

Если есть — его надо удалить:

''sudo rm /usr/bin/firefox''

Создать новый симлинк:

''cd /usr/bin/ && sudo ln -s /opt/firefox/firefox''

==== Запустить браузер из консоли ====

Так можно сразу распознать вероятные проблемы. 

Я запускаю его с диалогом выбора профиля. В принципе он должен загрузиться с профилем по-умолчанию, но у меня уже есть сохраненные профили и запускать «голый» браузер мне незачем:

''/opt/firefox/firefox -p''

Добавить профили, запустить.

Вероятные проблемы:

	XPCOMGlueLoad error for file /opt/firefox/libxul.so:
	libdbus-glib-1.so.2: cannot open shared object file: No such file or directory
	Couldn't load XPCOM.

		Лечение: ''sudo aptitude install libdbus-glib-1-2''
	или 

	./firefox: error while loading shared libraries: libstdc++.so.6: cannot open shared object file: No such file or directory
	Chances are that you are trying to install the 32 bit version of firefox in a 64 bit Debian environment.
	You should probably install '''apt install ia32-libs ia32-libs-gtk'''

	Установить нужные компоненты.

===== Добавить Firefox в Меню приложений =====

==== Сделать иконку ====

Скачать из сети картинку логотипа файл условно "firefox.png". Размер любой, но понадобится не больше 62 пикселей, поэтому громадные картинки не нужны.

Положить его куда-нибудь, где он будет находиться постоянно. Возможно, каталог ''/opt/firefox/icons/'' для этого лучшее решение (создать, если таковой отсутствует).

''sudo mv firefox.png /opt/firefox/icons/''

==== Создать файл „firefox.desktop” ====

Положить его куда-нибудь, где его ничего не тронет и откуда он будет подтянут в общее меню приложений.

''mcedit ~/.local/share/applications/firefox.desktop''

Вставить в него условно такой набор букв:

''[Desktop Entry]''
''Categories=Network;WebBrowser;''
''Comment=/opt''
''Exec=/opt/firefox/firefox -p'' __''astenix''__ ''%u''
''Icon=/opt/firefox/icons/firefox.png''
''MimeType=text/html;application/xhtml+xml;x-scheme-handler/http;x-scheme-handler/https;''
''Name=Firefox''
''StartupNotify=true''
''Terminal=false''
''Type=Application''

[F2] > [Esc] > [Esc]

==== Прописать файлу атрибуты ====

''chmod 644 ~/.local/share/applications/firefox.desktop''

Если надо запускать приложение через файл *.desktop из Dolphin, тогда файл надо сделать rwx (744), но для файлов в каталоге ''applications/'' это не нужно.

=== Сделать ярлык для запуска — для всех пользователей ===

Для этого “As a root” надо будет создать файл ''firefox.desktop'' в ''/usr/share/applications/'':

''sudo touch /usr/share/applications/firefox.desktop''

Если нужен ярлык только для одного юзера, то можно сразу создать его в ''~/.local/share/applications''

==== Назначить файл «представителем браузера» ====

Надо настроить внутренние механизмы KDE, иначе он начнет постоянно создавать в ''applications'' «свой» ярлык для отдельного профиля браузера (вроде ''~/.local/share/applicationsuserapp-Firefox-ENGLQ3.desktop'') и упрямо считать его «основным браузером». 

Простейшее решение — удалить файл со всеми настройками вроде «//*.mkv открывать через mpv//», что настраивается в 

''Параметры системы''
''> Приложения и окна''
''> Приложения по умолчанию''
''> Приложения по умолчанию''

и в 

''Параметры системы''
''> Приложения и окна''
''> Приложения по умолчанию''
''> Привязки файлов''

Соппсно:

''rm ~/.config/mimeapps.list''

''xdg-settings set default-web-browser firefox.desktop''

'''
kwriteconfig5 --file kdeglobals --group General --key BrowserApplication firefox.desktop
'''

Проверка:

''xdg-settings get default-web-browser''

Ждем в ответ:

''firefox.desktop''

Проверить поочередно. Все эти команды должны вернуть ''firefox.desktop'':

''xdg-settings get default-web-browser''
''xdg-mime query default text/html''
''xdg-mime query default x-scheme-handler/http''
''xdg-mime query default x-scheme-handler/https''

Заглянуть в

''Параметры системы''
''> Приложения и окна''
''> Приложения по умолчанию''
''> Привязки файлов''
''> text''
''> html''

Первым приложением для открытия html должен быть Firefox.

Удалить «мусорные» файлы запуска, если есть:

''rm -f ~/.local/share/applications/userapp-Firefox-*.desktop''

Если в системе остаётся установленным Firefox ESR, то его .desktop-файл (''firefox-esr.desktop'' или что-то аналогичное) тоже лучше удалить или переименовать, иначе KDE может снова предлагать его как альтернативный браузер.

А если при запуске браузера появится вопрос «Установить его браузером по-умолчанию?» — нажать отмену, потому что браузер и так уже по-умолчанию повсюду назначен, и теребить внутренние механизмы KDE уже незачем.

===== Установить новую версию браузера =====

Сперва удалить существующие файлы:

''rm -rf /opt/firefox*''

''rm /usr/bin/firefox''

Затем устанавливать новое.

===== Бэкап профиля =====

Проверить в настройках браузера, какой именно профиль используется:

	1. в строке поиска ''about:profiles''
	2. профилей может быть несколько, один из них помечен как «Профиль по умолчанию = Да».

По-умолчанию профили создаются и хранятся в ''/~/.mozilla/firefox/'', но всегда есть возможность выбора места хранения.

Рекомендуется регулярно бэкапить содержимое ''/~/.mozilla/firefox/''

См. настройки [[Software:Firefox]]
