====== Zensical ======

[ @debian @install @zensical ]

Следуем https://zensical.org/docs/get-started/

===== Установить =====

==== в систему ====

''pipx install --python /usr/bin/python3 zensical''

Всё будет установлено в каталог ''~/.local/share/pipx/venvs/zensical/''

==== в проект ====

Подразумевается, что в корневом каталоге проекта уже есть (или будет) каталог ''docs/''.

Перейти в корневой каталог проекта и

''zensical new .''

Появятся следующие файлы:

''.''
''├─ .github/''
''├─ docs/''
''│  ├─ index.md''
''│  └─ markdown.md''
''└─ zensical.toml''

===== Запуск сервера =====

''zensical serve''

 Открыть в браузере ''http://localhost:8000''

===== Базовая настройка =====

[project]
site_name = "My site"

Unless you're building documentation for offline usage, it's strongly recommended to specify the site_url setting as well, since it's a prerequisite for the following features:

	Instant navigation
	Instant previews
	Custom error pages


===== Собрать сайт из md =====

''zensical build''

===== Установить плагины =====

''pipx inject zensical mkdocstrings[python] mkdocs-material mkdocs-awesome-pages-plugin mkdocs-add-number-plugin mkdocs-graphviz mkdocs-mermaid2-plugin''

Тут inject, потому что плагины ВПИХИВАЮТСЯ в файлы mkdocs, и если позже надо будет посмотреть список установленного ПО в окружение pipx… 

…а для этого надо сперва активировать виртуальное окружение

''source ~/.local/share/pipx/venvs/mkdocs/bin/activate''

и затем можно проверить установленный софт, которые касается MkDocs:

''python -m pip list | grep mkdocs''

Удалить отдельный плагин, например ''mkdocs-awesome-nav'', надо выполнить в том же виртуальном окружении:

''pipx uninject mkdocs mkdocs-awesome-nav''

Через плагин ''material'' McDocs поднимет сайт c динамическими меню, поиском, ToC, табами на странице, mobile first и много другое (разные плагины).

Проверить установку:

''mkdocs --version''

Выход из виртуального окружения:

''deactivate''
