====== MkDocs ======

[ @debian @install @mkdocs ]

Его можно установить на уровне системы для всех проектов. Но сперва надо учесть версию. Проект MkDocs развился кардинально, его версия 2.0 представляет собой абсолютно новый проект, в котором не будут работать все привычные плагины — см. https://squidfunk.github.io/mkdocs-material/blog/2026/02/18/mkdocs-2.0/

Отпадает вся интеграция со всеми существующими плагинами; нет никакой возможности «легкой миграции данных»; надо переехать с yaml на toml; проект ещё в разработке, поэтому он ещё не production ready и баг-репорты не принимаются.

Решение: фриз обновлений и продолжать жить на ''MkDocs 1.0''. Какое-то время всё будет работать как надо, со временем появятся вероятные баги и надо будет как-то переезжать, или на 2.0, или на другой механизм автосборки документации проекта в общем доступе.

Поэтому здесь и далее рассматривается глобальная установка ''MkDocs 1.6.1'' в систему через ''pipx'', затем его принудительная заморозка. Сам проект, конечно же, надо запускать из виртуального окружения.

Учесть, что какие-то плагины тоже могут отмереть сами по себе (например ''mkdocs-graphviz-plugin'').

Если ранее MkDocs уже был установлен, его надо удалить:

''pipx uninstall mkdocs''

Установить MkDocs с конкретной версией:

''pipx install --python /usr/bin/python3 mkdocs==1.6.1''

Всё будет установлено в каталог ''~/.local/share/pipx/venvs/mkdocs/'' 

Если после этого даже случайно не делать 

''pipx upgrade mkdocs''

то и вероятные обновления не подтянутся.

Установить нужные (мне) плагины:

''pipx inject mkdocs mkdocstrings[python] mkdocs-material mkdocs-awesome-pages-plugin mkdocs-add-number-plugin mkdocs-graphviz mkdocs-mermaid2-plugin''

Тут inject, потому что плагины ВПИХИВАЮТСЯ в файлы mkdocs, и если позже надо будет посмотреть список установленного ПО в окружение pipx… 

Можно добавить плагин ''mkdocs-awesome-nav'' , но учесть, что он конфликтует с ''awesome-pages'', и если пойти дальше с ''awesome-pages'', то ''mkdocs-awesome-nav'' лучше удалить.

…а для этого надо сперва активировать виртуальное окружение

''source ~/.local/share/pipx/venvs/mkdocs/bin/activate''

и затем можно проверить установленный софт, которые касается MkDocs:

''python -m pip list | grep mkdocs''

Удалить отдельный плагин, например ''mkdocs-awesome-nav'', надо выполнить в том же виртуальном окружении:

''pipx uninject mkdocs mkdocs-awesome-nav''

Через плагин ''material'' McDocs поднимет сайт c динамическими меню, поиском, ToC, табами на странице, mobile first и много другое (разные плагины).

Проверить установку:

''mkdocs --version''

Выход из виртуального окружения:

''deactivate''
