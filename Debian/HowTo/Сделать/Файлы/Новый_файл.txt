====== Новый файл ======

[ @console ]

Удивительно, но в Linux нет определенной команды для создания новых файлов. Все пути — обходные.

===== Стандартный способ =====

''> 123.txt''

Это перенаправление вывода из консоли в файл. Вывод пустой, поэтому просто будет создан пустой файл. 

Можно создать файл сразу с текстом

''echo Это текст в первой строке > 123.txt''

Опытные юзеры всегда будут обрамлять вывод echo кавычками:

''echo "Это текст в первой строке" > 123.txt''

Можно создать сразу каталог и файл в нём:

''mkdir -p Work; echo "Это текст в первой строке" > Work/January123.txt''

Можно перенаправить в файл не текст, а вывод какой-либо команды:

''date > CurrentDate.txt''

===== Через “touch” =====

На самом деле команда ''touch'' должна всего лишь «потрогать» существующий файл и обновить время последнего доступа к нему, якобы «файл только что открывали», и //если// указанный файл отсутствует, то он будет создан (иначе трогать нечего):

''touch 123.txt''

Чтобы новые файлы из-за ''touch'' не создавались, надо писать перечень трогаемых файлов с ключом ''-c''.

''touch -c 123.txt''

Через touch можно одной командой создать сразу много файлов, указав их имена последовательно:

''touch 123.txt 345.doc hello.js viva.txt''

То есть, создание файлов через ''touch'' это косвенный эффект. Но многие считают его основным.
