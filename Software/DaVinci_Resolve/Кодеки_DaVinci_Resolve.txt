====== Кодеки DaVinci Resolve ======

[ @davinci @theory @video ]

===== Основные кодеки для видео =====

1. [[+DNxHD]] и **DNxHR** — используются в DaVinci Resolve как основные. DNxHD используется для видео в формате HD, а DNxHR — для более высоких разрешений (4K и более). Оба поддерживают хорошее сжатие с потерями, сохраняя высокое качество изображения.
2. **ProRes: Apple ProRes** — один из стандартов в индустрии для монтажа и для обработки видео в высоком качестве. DaVinci Resolve поддерживает профили ProRes 422 (и его вариации), ProRes 4444 и ProRes 4444 XQ.
3. **H.264** и **H.265/HEVC —** кодеки для сильного сжатия видео при сохранении хорошего качества. H.264 является стандартом для видео на YouTube, а HEVC (H.265) предоставляет лучшее сжатие с меньшими размерами файлов при схожем качестве.
4. **AV1** — свободный кодек, основной в проектах DaVinci Resolve.
5. **CineForm** — тоже профессиональный кодек с высококачественным сжатием, который поддерживается DaVinci Resolve, особенно в кино- и телеиндустрии.
6. **XAVC/XAVC-S** — формат видео с камер Sony.

===== Кодеки для аудио =====

1. **PCM** (файлы WAV и AIFF) — некомпрессированный аудиоформат.
2. **AAC** — кодек для сжатого аудио в контейнерах, таких как MP4, и является стандартом для потокового аудио. Но он проприетарный, по-умолчанию недоступен.
3. **Dolby Audio** — AC-3 и Dolby Atmos, многоканальный звук. По-умолчанию недоступен.

===== CUDA и OpenCL =====

CUDA (Compute Unified Device Architecture) — параллельная вычислительная архитектура и программная платформа, разработанная компанией NVIDIA для выполнения вычислений на графических процессорах (GPU). Вкратце называется «технология GPU-ускорения для обработки видео». В отличие от традиционных CPU, которые работают с ограниченным количеством потоков (ядра), GPU оснащены тысячами ядер, что позволяет выполнять множество операций одновременно. 

1. CUDA: Для карт Nvidia.
2. OpenCL: Для других GPU, включая карты AMD.

===== Кодеки для монтажа =====

1. **Optimized Media** — прокси-файлы с пониженным разрешением и сжатыми кодеками, которые используются для более быстрого монтажа и работы с видео. Они могут быть использованы для видео, снятого в формате ProRes, DNxHD или других профессиональных форматах, что позволяет работать с большими проектами без перегрузки системы.
2. **Render Cache** — временные файлы, которые Resolve создает для ускорения рендеринга при монтаже. Эти файлы могут быть сжатыми (например, в формате ProRes или DNxHR) для ускоренной обработки.

===== Контейнеры видео =====

1. **MOV** — для хранения видео в ProRes и DNxHD.
2. **MXF** — часто используется в телевизионной и киноиндустрии
3. **MP4** — обычно используется для видео в H.264 или H.265.
4. **AVI** — старый, привычный формат видео.

===== Специальные кодеки =====

1. **Blackmagic RAW** — собственный формат компании Blackmagic Design для видео с камер Blackmagic. Он поддерживает различные уровни сжатия для сохранения баланса между качеством и размером файла.
2. **REDCODE RAW** (R3D) — формат для видео с камер RED, который поддерживает рабочие процессы с высоким динамическим диапазоном и высоким качеством изображения.

===== Как DaVinci Resolve работает с кодеками =====

Неоднозначно.

В Linux (почти) всё, что связано с видео, работает через FFmpeg, поэтому, кагбэ, должно было бы… Но DaVinci Resolve в своем ядре FFmpeg напрямую не использует и опирается на свои собственные алгоритмы и API для работы с кодеками:
* платформенные кодеки: DaVinci Resolve использует стандартные системные кодеки на каждой платформе (Windows, macOS, Linux). Например, macOS может использовать встроенные кодеки Apple для видео (H.264, H.265, ProRes), а Windows — кодеки, предоставляемые сторонними разработчиками.
* внешние кодеки: В некоторых случаях DaVinci Resolve может использовать кодеки сторонних производителей, такие как ProRes или DNxHD, для работы с видеофайлами в редакторе.
* интерфейсы для декодирования: DaVinci Resolve использует различные внутренние механизмы и интерфейсы для работы с различными форматами и кодеками, но это не обязательно связано с FFmpeg напрямую.
