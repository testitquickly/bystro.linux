====== Kate ======

[ @kate @fonts ]

Базовый текстовый редактор в KDE, предустановлен. В норме выглядит просто, с плагинами в прямых руках превращается в резвый IDE.

===== Шрифты =====

* Ysabeau
* IBM Plex Mono Text
* Hack 
* Nimbus Sans L 
* Noto Mono 
* Source Code Pro 

===== Подсветка текста =====

Предположим, что я держу код для Bash в файлах с расширением .var или .fn. Надо указать Kate, что их содержимое надо подсвечивать так же, как файлы .sh

''Настройка''
''> Настроить Kate''
''> Открытие и сохранение''
''> Типы файлов''

Тип файла = //Языки сценариев Bash//
Расширение файлов = добавить в перечень расширений свои в формате ''*.var;''

Целиком это может выглядеть так (видно, что можно указывать полное название отдельных файлов):

''*.sh;*.bash;*.var;*.fn;.bashrc;.bash_profile;.bash_login;.profile''

Подсветка применится после открытия файла с расширением из этого перечня.

===== Перенос строк =====

Есть множество строк вроде 

'''02.12.2024 16-47.vtt'''
'''02.12.2024 17-03.vtt'''
'''02.12.2024 19-10.vtt''' 

надо превратить их в один ряд:

'''02.12.2024 16-47.vtt' '02.12.2024 17-03.vtt' '02.12.2024 19-10.vtt'''

Ctrl+R — переключить в режим “Регулярное выражение”.

“Искать”: 

''\n''

“Заменить на”: оставить пустым.

Но если перед каждой строкой будет пробел, тогда замену надо делать через ''\s*\n''

===== Упорядочить строки по алфавиту =====

''Ctrl+Shift+S''

Но лучше сперва проверить

1. Выделить строки, которые надо отсортировать
2. Пройти:
''Сервис''
''> Сценарии''
''> Редактирование''
''> Упорядочить текст в естественном порядке (Ctrl+Shift+S)''

==== **Естественный порядок** ====

Числа внутри строк трактуются как числа, а не текст:

* a1
* a2
* a10

Заглавные буквы идут раньше строчных:

* A
* B
* a

Латиница перед кириллицей:

* a
* b
* а
* б

==== **Алфавитный порядок** ====

Сравнение идёт символ за символом; учитываются ASCII/Unicode коды; результат сортировки машинно-корректный, но раздражающий:

* a1
* a10
* a2
