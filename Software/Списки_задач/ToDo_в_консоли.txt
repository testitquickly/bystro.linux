====== ToDo в консоли ======

[ @console @todo ]

Придумал простой способ добавить самому себе напоминалки о чём-то, что нужно сделать, глядя в консоль — прямо в консоль.

Тут нет проектов, дат, группировки, приоритета и что там ещё может быть, это всё есть в [[Taskwarrior]] и [[Todoman]].

===== Основная идея =====

Можно выводить в консоли содержимое какого-то файла.

1. Сделать текстовый файл:

''mcedit ~/todo.txt''

2. Добавить в него одну и более строк с задачами. Обязательно добавить одну пустую строку в конце, это нужно для грамотного вывода содержимого файла в консоль.

3. Отредактировать файл ''.bashrc''

''mcedit ~/.bashrc''

В самую последную строку этого файла добавить вызов содержимого из ''todo.txt'':

	''# Вывод содержимого в консоль''
''cat ~/todo.txt''

Теперь при открытии каждого окна Konsole (или при логине под моим юзером в консоль в другом tty; или при каждом перечитывании «''source ~/.bashrc''») первые строки вывода в консоли будут заполнены содержимым ''~/todo.txt''

Можно добавлять в него строки прямо из консоли:

''echo 'Положить диафильмы в коробку' >> ~/todo.txt''

 ''>'' — уберет из файла всё содержимое и добавит в него новую строку
 ''>>''  — оставит содержимое файла и добавит новую строку в конце файла

Текст для ''echo'' можно не обрамлять кавычками, но лучше да, чтобы не нарушать здоровые привычки.

И можно, не заходя в файл, удалить, например, третью строку:

''sed -i '3d' ~/todo.txt''

В принципе норм, но каждый раз надо всё это аккуратно набирать, не ошибаясь в символах, особенно в разнице между ''>'' и ''>>''.

===== Автоматизация команд =====

Можно превратить все эти команды в функции. 

Сделать в каталоге со всеми скриптами файл:

''mcedit myTodo.fn''

Вставить:

{{{code: lang="sh" linenumbers="True"
    # это файл с функциями, тут шебанг не нужен
    # комментарий с восклицалкой #! отменяет вывод функции
        # в скрипте “displayVariables”

export myToDoFile=$HOME/todo.txt

#! Добавить строку в файл todo.txt
todoAddTask() {
    # Проверить аргументы для команды,
    # чтобы не создавать пустые строки
    if [ $# -eq 0 ]; then
        cat "$myToDoFile"
        return
    fi

    # временный файл (нельзя писать в тот же файл, из которого читаем)
    local tmpfile=$(mktemp)

    # добавить новую строку,
    # выполнить естественно-алфавитную сортировку строк
    # записать результат во временный файл
    {
        cat "$myToDoFile"
        printf '%s\n' "$*"
    } | sort -V > "$tmpfile"

    # передать содержимое временного файла в оригинальный
    cat "$tmpfile" > "$myToDoFile"

        # показать результат в консоли
    todoDisplayTask
}

#! Показать в консоли содержимое todo.txt
#! и добавить перед каждой строкой порядковый номер
todoDisplayTask() {
        # проверить, что файл есть
    [ -f "$myToDoFile" ] || return
        #  очистить консоль
    clear
        # добавить пустую строку
    echo " "

     # подсчитать количество строк,
    # вывести их номера в формате 1)
    # и добавить “пробел выравнивания“ после скобок для 1)-9)
    #nl -w1 -s') ' "$myToDoFile" | sed -E 's/^([1-9]\)) /\1  /'

    # подсчитать количество строк (через nl),
    # вывести их номера в формате 1)
    # и добавить “пробел выравнивания“ перед числами 1)-9) через
    # встроенные возможности nl
    nl -w2 -s') ' "$myToDoFile"

        # добавить пустую строку
    echo " "
}


#! Удалить строку из файла todo.txt
todoRemoveTask() {
        # проверить аргумент — у команды td- обязательно должен быть номер строки
    if ! [[ "$1" =~ ^[0-9]+$ ]]; then
        cat "$myToDoFile"
        echo -e "\nУказать номер строки, которую надо удалить\n"
        return
    fi

        # защита от обработки пустого файла
        # если файл пустой — ничего не делать
    if [ ! -s "$myToDoFile" ]; then
        return
    fi

        # подсчитать количество строк
    local total_lines
    total_lines=$(wc -l < "$myToDoFile")

        # проверить диапазон строк (если в строке 10 строк, а я командую удалить 28-ю…)
    if [ "$1" -lt 1 ] || [ "$1" -gt "$total_lines" ]; then
        echo -e "\n\tТакой строки в todo.txt нет\n"
        return
    fi

    local tmpfile
    tmpfile=$(mktemp)

        # сделать копию оригинального файла во временный
    cat "$myToDoFile" > "$tmpfile"

        # редактировать временный файл:
        # удалить нужную строку
        # и вернуть в исходный файл обновленное содержимое
    sed "${1}d" "$myToDoFile" > "$myToDoFile.tmp" && cat "$myToDoFile.tmp" > "$myToDoFile"

        # показать результат в консоли
    todoDisplayTask
}

}}}

Отредактировать файл ''~/.bashrc''. Там должна быть объявлена ссылка на каталог со скриптами, например:

''export MY_SCRIPTS="$HOME/myScripts"'' 

Добавить в конец файла ''~/.bashrc'': 

{{{code: id="todo_in_console" lang="sh" linenumbers="True"
# -------------------------
# --- ToDo в консоли
# -------------------------

    # переменные и названия функций из файла
source $MY_SCRIPTS/myTodo.fn

alias tdedit='mcedit $myToDoFile' # редактировать todo.tx    # Алиасы для обхода ошибок при наборе

for i in td ев; do alias $i='todoAddTask'; done # “td текст”; добавить задачу в myToDo

for i in td- tdr ев- евк; do alias $i='todoRemoveTask'; done # “td- 4” = удалить задачу номер четыре

    # показать список задач при каждом открытии терминала
todoDisplayTask # показать содержимое todo при открытии Konsole

}}}

Сохранить, выйти, перечитать содержимое файла:

''source ~/.bashrc''

Теперь каждый раз когда открывается окно терминала, будет выполнена функция ''todoDisplayTask''.

==== Добавить задачу ====

''td Положить диафильмы в коробку''

И даже кавычки не нужны.

==== Удалить задачу ====

Ориентируемся по номерам строк. Например, удалить задачу №2:

''td- 2''

===== Ограничения =====

Нельзя передавать этому скрипту где-либо в строке символ „''*''” → он принудительно вставляет в файл вызов команды ''ls''

И вот список символов, которые можно, но незачем использовать:

''`''
''~''
''#''
''&''
''*''
''(''
'')''
'';''
'''''
''"''
''\''

А вот эти вполне можно, хотя тоже незачем:

''!''
''@''
''$''
''%''
''^''
''/''
''=''
''_''

===== Безопасность файла Туду =====

Имеет смысл держать файл с Туду в Dropbox. 

В принципе можно в файле ''~/.bashrc''  указать ''myToDoFile=$HOME/Dropbox/todo.txt''

Но я предпочитаю постоянно видеть ''todo.txt'' в каталоге профиля. Поэтому: 

* положить файл ''todo.txt'' в Dropbox
* в каталоге своего профиля сделать на него симлинк

''ln -s $HOME/Dropbox/todo.txt''
