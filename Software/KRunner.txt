===== Запуск KRunner =====

По-умолчанию KRunner запускается по Alt+F2. Но бывают исключения.

''Параметры системы''
''> Рабочая среда''
''> Комбинации клавиш''
''> Приложения''
''> «Открыть строку поиска и запуска»'' 

Назначить сочетание клавиш для «Открыть строку поиска и запуска KRunner». По-умолчанию ''Alt+Space''. Удобнее ''Alt+Esc''.

Вроде бы то же самое настраивается в разделе «Системные службы», но на самом деле нет.

===== Выполнение алиасов из .bashrc =====

Предположим, у меня есть алиас '''dv''' (см. [[Debian:HowTo:Сделать:Список алиасов из .bashrc]]). 

Можно сделать скрипт, который 

1. откроет новое окно Konsole
2. в этом окне выполнит указанную мною команду
3. и оставит окно Konsole открытым

Этот скрипт можно вызвать через KRunner и передать ему в виде параметра команду/алиас, которую надо выполнить.

1. Сделать скрипт:

''cat > /home/mySettings/myScripts/konsole_run_command.sh <<'EOF'''
''#!/bin/bash''
''konsole -e bash -ic "$*; exec bash" # выполнить команду и оставить окно Konsole открытым''
''EOF''

Сделать его исполняемым:

''chmod +x /home/mySettings/myScripts/konsole_run_command.sh''

Сделать симлинк '''qq''' на этот файл в /bin профиля юзера:

''ln -s /home/mySettings/myScripts/konsole_run_command.sh ~/.local/bin/qq''

2. Вызвать окно KRunner

''Alt+Space''

Набрать команду:

''qq dv''

==== Решение проблемы со вторым перечитыванием .bashrc ====

У этого подхода есть мелкий недостаток: содержимое файла  ''~/.bashrc'' будет прочитано два раза: перед инициализацией окна Konsole и после выполнения аргумента/команды. В этом есть смысл — чтобы выполнить вызываемую команду, надо сперва прочитать  ''~/.bashrc''. Но если после выполнения вызванной команды надо оставить открытым окно Konsole, в открытую оболочку bash надо передать содержимое  ''~/.bashrc'', а для этого его надо снова перечитать. Если в этом файле есть команда, которая что-то выводит в консоль — [[Списки задач:ToDo в консоли]] — то в одном окне произойдет двойной вывод данных.

Это поведение можно подавить: в файле  ''~/.bashrc'' команду, которая выводит в консоль текст при каждой инициализации командной оболочки bash, надо обернуть в особе условие.

''mcedit ~/.bashrc''

Обернуть нужную команду в IF:

''if'' [[-z "$SKIP_COMMAND_ON_LOADING_BASH"| ''-z "$SKIP_COMMAND_ON_LOADING_BASH"'' ]]''; then''
	''cat $myToDoFile # команда''
''fi''

Отредактировать скрипт:

''mcedit /home/mySettings/myScripts/konsole_run_command.sh''

Вставить это:

{{{code: lang="sh" linenumbers="True"
#!/bin/bash

# Выполняет команду в Konsole, оставляет окно открытым
# и позволяет пропускать определённые команды при втором shell

konsole -e bash -ic '
    # Выполнить команду, которая была передана скрипту
'"$*"';

    # и установить флаг для пропуска команды во втором shell
export SKIP_COMMAND_ON_LOADING_BASH=1

    # заново инициализировать интерактивный shell. Файл .bashrc
    # будет снова прочитан, но на этот раз команда, которая стоит
    # за условием SKIP_COMMAND_ON_LOADING_BASH
    # не будет выполнена
exec bash
'

}}}

Сохранить, выйти. 

Теперь при исполнении команды через KRunner > ''qq'' __''command''__ будет открываться окно Konsole, в котором будет выполнена вызванная команда, и окно Konsole будет оставаться открытым, и все алиасы из ''~/.bashrc'' в этом окне будут полностью доступны.

==== Настроить поиск файлов в KRunner ====

Работает на основе [[Baloo]],  и в принципе здраво этим всем НЕ пользоваться, но если очень хочется, то ок.

''Параметры системы'' 
''> Рабочая среда''
''> Поиск''
''> Поиск файлов''
''> "Включить службу поиска файлов"''

Указать папки, в которых не надо индексировать файлы. В терминале команда balooctl status проверяет саму службу индексирования

''Параметры системы''
''> Рабочая среда''
''> Поиск'' 
''> Поиск в Plasma''
