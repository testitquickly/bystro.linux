====== Калькуляторы ======

[ @console @qalculate @speed_crunch @kcalc @bc @casio ]

===== Qalculate! =====

https://qalculate.github.io/

Интерфейс ближе к KCalc, и видна последовательность ввода, но он однострочный. Чтобы увидеть всю историю, надо кликнуть по отдельной кнопке.

''2+2*2=6''

Внутри по-умолчанию используется адаптивный анализ, ''234+234×999+77'' превращается в ''234+(234×999)+77''. 

	В режиме «Цепной синтаксис» ''2+2*2=6'' превращается в ''(2+2)*2=8''.

Переменные поддерживает, можно переключиться между видом «дроби» и «числа с запятой».

Расчет процентов работает as expected, ''22+10%=24,2''.

Можно задавать переменные и использовать их в расчетах:

''val1 = 1''
''val2 = 2''
''val1 + val2 = 3''

Шрифты настраиваются очень вольготно.

''sudo aptitude install qalculate-gtk''

См. настройки: [[Qalculate!]]

===== SpeedChrunch =====

https://github.com/sanchahar/speedcrunch

Сразу очень хорош — такой себе терминал. Видно всё пошагово, можно пролистывать и переиспользовать ввод из истории. 

''2+2*2=6''

Можно задавать переменные и использовать их в расчетах:

''val1 = 1''
''val2 = 2''
''val1 + val2 = 3''

Недостаток: в нём **очень** сложно, слишком уж «по-научному» делается расчёт процентов. Нельзя просто ввести „''22 + 10% = 24,2''”, надо вводить „''22 + (22 * 10 / 100) = 24,2''”

''sudo aptitude install speedcrunch''

===== KCalc =====

''kcalc''

Хорош в своей простоте, но не отображает последовательность вводимых чисел, отрабатывает одну операцию за раз. И иногда откровенно глючит — напрягает непредсказуемостью.

===== bc =====

Консольный «калькулятор». Точнее, программа для ускорения расчетов в консольном окружении, которую можно использовать в своих программах.

	'bc' is an arbitrary precision calculator language that supports arbitrary precision numbers with interactive execution of statements. 

	There are some similarities in the syntax to the C programming language. 

	A standard math library is available by command line option. If requested, the math library is defined before processing any files. 

	bc starts by processing code from all the files listed on the command line in the order listed. After all files have been processed, bc reads from the standard input. All code is executed as it is read. (If a file contains a command to halt the processor, bc will never read from the standard input.)

Запуск: 

''bc''

Выход: 

''quit''

Позволяет выполнять расчеты на манер древних компьютеров с Basic, вроде 

''2+2*2'' 

= «6», а также 

''sqrt (25)''

= «5»

===== На столе =====

Иногда на экране слишком много окон, между которыми надо переключаться. В этом случае здорово помогает держать фокус реальный калькулятор на столе. Идеальным настольным калькулятором хотелось бы назвать Casio «fx-991EX», но он не идеальный.

По сравнению с привычными однострочными калькуляторами его экран типа „Natural Textbook Display” поражает красотой. Там буквально отображаются цифры из шрифта SchoolBook — стандарт школьных учебников. Там дроби, время, можно передвигать курсор, там много всякого.

Там можно даже создавать экселеподобные таблицы с расчетами «на лету», матрицы, векторные вычисления — всего девять режимов. Вчитаться в инструкцию придётся.

Неудобно то, что ввод расчётов происходит только на одной строке, и если ввод длинный, то начинается пролистывание влево-вправо.

Через пять-десять минут бездействия настольный калькулятор полностью отключается, и набиваемые в нём таблицы не сохраняются. После компьютера к этому привыкнуть почти невозможно.

Есть его более новый собрат — Casio «FX-991ES Plus». Там перепродумана и местами очень здорово упрощена последовательность нажимаемых кнопок для вызова разных режимов (желтый и голубой регистр), и даже можно поднапрячься и перепривыкнуть ко всему этому. Но у второго внезапно бесконечно некрасивый шрифт.
