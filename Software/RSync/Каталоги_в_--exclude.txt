====== Каталоги в --exclude ======

[ @rsync @exclude @folder ]

===== Относительный путь к исключаемым каталогам =====

Надо перенести содержимое ''/Документы'' в ''//NAS/Документы'', кроме содержимого нескольких каталогов:

''rsync -auzv \''
	''-e "ssh -i $ssh_key" \''
	''--remove-source-files \''
	''--skip-compress=$file_list \''
	__''--exclude="/home/hub/Документы/Mind Maps/" \''__
	__''--exclude="/home/hub/Документы/Автомобиль/_CarteaVerde/" \''__
	''"/home/hub/Документы/" "$nas_enc_docs"''

Но в этом случае всё содержимое каталогов, которые //должны// быть проигнорированы, ВНЕЗАПНО будет полностью перенесено в NAS. Есть неочевидный момент — RSync рассматривает только относительные пути к каталогам под параметром ''--exclude'', поэтому перечисленные тут каталоги RSync //видит// как ''/home/hub/Документы//home/hub/Документы/Mind Maps/'', а такого каталога нет, поэтому команда воспринимается как корректно отработанная и скрипт продолжает работу…

Правильно так: 

''rsync -auzv \''
	''-e "ssh -i $ssh_key" \''
	''--remove-source-files \''
	''--skip-compress=$file_list \''
	__''--exclude="Mind Maps/" \''__
	__''--exclude="Автомобиль/_CarteaVerde/" \''__
	''"/home/hub/Документы/" "$nas_enc_docs"''

В этом случае RSync //увидит// каталог ''/home/hub/Документы/Mind Maps/'' и послушно проигнорирует его.

Ещё надо учесть, что каталог (со всем своим содержимым) в exclude надо добавлять с закрывающим слэшем, тогда RSync проигнорирует и каталог, и всё его содержимое. 

''--exclude="Mind Maps/"''

Если закрывающего слэша не будет, RSync проигнорирует сам каталог, но все файлы или подкаталоги внутри него будут включены в синхронизацию.

===== Проверить наличие каталогов =====

Учесть ещё одну особенность RSync — если какой-то из каталогов в ''--exclude'' исчезнет с диска, сообщений об этом не будет. 

Добавить файл для функций:

{{{code: lang="sh" linenumbers="True"
    # проверить наличие исключаемых каталогов; полный стоп при любой ошибке
fn_check_excluded_folders() {
    local root="$1"; shift
    local dirs=("$@")
    for dir in "${dirs[@]}"; do
        if [[ ! -d "$root/$dir" ]]; then
                # если что-то не так — показать алерт
            echo -e "\n\t${clr_RED_BRIGHT}Стоп!${clr_NONE} Проблема с каталогом ${clr_BLUE_BRIGHT}$root/$dir${clr_NONE}\n" >&2
                # полный стоп скрипта при любой ошибке
            exit 1
        fi
    done
}

    # создать массив исключаемых каталогов из пользовательского списка
fn_list_excluded_folders() {
		# первый аргумент — корневой каталог условно назовём «src»
	local src="$1"
		# сдвинуть позиционные параметры функции на один влево:
        # после shift первый аргумент (src) убирается из $@, и $@ остаётся только
        # с каталогами, которые нужно проверять и превращать в --exclude.
        # без shift в массив dirs попал бы весь список аргументов, включая путь
        # исходного каталога, и функция работала бы неправильно.
        # Не перемещать эту команду!
    shift
    local dirs=("$@")
		# очистить (создать пустой) массив
    local exclude_array=()

		# для каждого существующего каталога в списке исключаемых каталогов
		# добавить --exclude="$dir/".
    for dir in "${dirs[@]}"; do
        if [[ -d "$src/$dir" ]]; then
            exclude_array+=(--exclude="$dir/")  # формируем элемент массива
        fi
    done

         # вернуть массив построчно через printf для
         # безопасного чтения через readarray
    printf '%s\n' "${exclude_array[@]}"
}

copy_files_with_rename() {
    : 'Копирует файлы и каталоги из source_folder в dest_folder.
       Если файл или каталог с таким именем уже есть в dest_folder,
       добавляет таймcтамп _yyyy_mm_dd_h_m_s к имени.
       Исходные файлы и каталоги остаются на месте.'
    source_folder="$1"
    dest_folder="$2"

    mkdir -p "$dest_folder"

    # Перебираем всё в каталоге
    find -L "$source_folder" -mindepth 1 -maxdepth 1 | while IFS= read -r item; do
        # Получение имени без пути, /dir/a/file.jpg → file.jpg
        name=$(basename "$item")
        # Формирование целевого пути с указанием уже полученного имени файла
        # хак на случай двойного слэша в таргете — удалить лишний слэш
        target="${dest_folder%/}/$name"

        # Проверка существования файла в приёмнике
        # Если файл/каталог уже есть в $dest_folder, тогда формируем новое имя
        if [ -e "$target" ]; then
            # Генерация временной метки вида 2026_03_10-12_16_41
            timestamp=$(date +%Y_%m_%d-%H_%M_%S)
            # Формирование нового имени для файла с добавлением временной метки
            # добавлен хак на случай двойного слэша в таргете — удалить лишний слэш
            # ${name%.*} — имя без расширения.
            # ${name##*.} — расширение.
            target="${dest_folder%/}/${name%.*}-$timestamp.${name##*.}"
        fi


        # копируем файлы и каталоги по-разному
        # -d — проверка: существует ли и является ли каталогом.
        # $item — путь к объекту, который нашли через find
        if [ -d "$item" ]; then
            # Для каталогов добавляем слэш, чтобы копировалось
            # содержимое, а не сам каталог.
            rsync -azP "$item/" "$target/"
        else
            # Для файлов оставляем обычный вызов.
            rsync -azP "$item" "$target"
        fi

    done
}

move_files_with_rename() {
    : 'Перемещает файлы и каталоги из source_folder в dest_folder.
       Если файл или каталог с таким именем уже есть в dest_folder,
       добавляет таймcтамп _yyyy_mm_dd_h_m_s к имени.
       Можно указать список имён/каталогов для исключения через параметр --exclude.
       После перемещения удаляет исходные файлы и пустые каталоги.'

        # исходный каталог, откуда перемещаем
    source_folder="$1"
        # каталог назначения
    dest_folder="$2"
        # убираем первые два аргумента
    shift 2
        # список имён/каталогов для исключения, пустой по умолчанию
        # никогда не добавлять слэши к именам каталогов в этом списке
    exclude_list=""

    # Обрабатываем именованный параметр --exclude
    while (( "$#" )); do
        case "$1" in
            --exclude)
                exclude_list="$2"  # пробел-разделённые имена для пропуска
                shift 2
                ;;
            *)
                shift
                ;;
        esac
    done

    # Создаём каталог назначения, если его ещё нет
    mkdir -p "$dest_folder"

    # Перебираем только объекты первого уровня внутри source_folder
    # -L — следовать символическим ссылкам
    # -mindepth 1, -maxdepth 1 — только первый уровень (без рекурсии)
    find -L "$source_folder" -mindepth 1 -maxdepth 1 | while IFS= read -r item; do
        name=$(basename "$item")  # имя файла/каталога без пути

        # Проверка на исключение: если имя входит в список exclude_list, пропускаем
        for ex in $exclude_list; do
            [ "$name" = "$ex" ] && continue 2  # continue 2 — перескочить на следующий item
        done

        # Формируем целевой путь в dest_folder
        # ${dest_folder%/} — удаляем возможный слэш в конце
        target="${dest_folder%/}/$name"

        # Если объект с таким именем уже есть в dest_folder — добавляем таймштамп
        if [ -e "$target" ]; then
            # Генерация временной метки вида 2026_03_10-12_16_41
            timestamp=$(date +%Y_%m_%d-%H_%M_%S)
            # Формируем новое имя: name + таймштамп + исходное расширение
            target="${dest_folder%/}/${name%.*}-$timestamp.${name##*.}"
            # Для каталогов убираем расширение
            [ -d "$item" ] && target="${dest_folder%/}/${name}-$timestamp"
        fi

        # Копирование или перемещение объектов
        # -d — проверка: существует ли и является ли каталогом.
        if [ -d "$item" ]; then
            # Если каталог:
            # Добавляем слэш на конце источника, чтобы скопировалось содержимое, а не сам каталог
            # --remove-source-files — удаляет файлы после копирования, но каталоги остаются
            rsync -azP --remove-source-files "$item/" "$target/"
        else
            # Если файл — обычное копирование с удалением исходника
            rsync -azP --remove-source-files "$item" "$target"
        fi
    done

    # После перемещения удаляем пустые каталоги в исходном каталоге
    find "$source_folder" -type d -empty -delete
}

#echo -e "\n\n\t${clr_AMBER}Перенос${clr_NONE}\n"
#move_files_with_rename "$a/" "$b/" --exclude "123"

list_excluded_folders() {
        # создать пустой массив
    local exclude=()
        # принять параметры
    local src="$1"
        # сдвинуть позиционные параметры функции или скрипта на один влево:
        # После shift первый аргумент (src) убирается из $@, и $@ остаётся только с каталогами, которые нужно проверять и превращать в --exclude.
        # Без shift в массив dirs попал бы весь список аргументов, включая путь исходной папки, и функция работала бы неправильно.
        # Не перемещать эту команду!
    shift
    local dirs=("$@")

    for dir in "${dirs[@]}"; do
        if [[ -d "$src/$dir" ]]; then
            exclude+=(--exclude="$dir/") # обязательно закрывающий слэш
        else
                # показать алерт
            echo -e "\n\t${clr_RED_BRIGHT}Стоп! Отсутствует каталог '$src/$dir' ${clr_NONE}\n" >&2
                # и прекратить выполнение скрипта
            exit 1
        fi
    done

    # возвращаем массив через echo
    echo "${exclude[@]}"
}

}}}

В файле со скриптом синхронизации:

{{{code: lang="sh" linenumbers="True"
#!/bin/bash
set -euo pipefail

    # Если надо скопировать сам каталог — без закрывающего слэша.
    # Если надо скопировать содержимое каталога — нужен закрывающий слэш.

    #-a (или --archive): Архивный режим, включает рекурсивную копию (-r) и сохранение прав доступа, владельцев, меток времени (-ptgoD)
    #-u (или --update): Пропустить на получателе файлы, которые уже более свежие, чем пришло от отправителя. Очень помогает не затереть новые правки.
    #-z: Сжимает данные при передаче, экономит трафик и ускоряет копирование по сети,
        # но вредит процессору при копировании
        # музыки (.mp3, .ogg, .flac)
        # фото (.jpg, .png, .webp)
        # видео (.mp4, .mkv)
        # архивов (.zip, .tar.gz, .7z)
        # и некоторые файлы pdf
        # Это уже сжатые файлы, сжимать их снова нет смысла, а иногда это увеличивает их объем. Для их игнорирования задана опция --skip-compress
    # -v (--verbatim): показывать файлы, которые задействованы в копировании
    # -P: отображается прогресс копирования + возможность докачки, если произошел обрыв соединения
    # -R (или --relative): нужен, если в один бэкап-каталог сохраняем сразу несколько каталогов из разных мест на диске.
        # При одиночной команде копирования не нужен.
    # -e ssh: заюзать ssh
    # --delete: ОСТОРОЖНО! Удаляет с NAS файлы, которые уже были в бэкапе, но были удалены на ноутбуке. На NAS останется только то, что есть на ноуте.
    # --delete-delay: чуть безопаснее — файлы удаляются во время их передачи, а не в её начале.
    # --remove-source-files: ОСТОРОЖНО! Переносит файлы с ноута на NAS (каталоги остаются).

    # ПЕРЕМЕННЫЕ
# -------------------------------------------- #

    # глобальные переменные: nas_ip, nas_host, ssh_key…

file_with_variables="/home/myScripts/variables.var"
file_with_functions="/home/myScripts/nas/rsync/functions.fn"

    # проверить наличие служебных файлов, полный стоп при любой ошибке
    # эту функцию надо держать только в этом файле
fn_check_service_files() {
    if [[ ! -f "$file_with_variables" || ! -f "$file_with_functions" ]]; then
        echo -e "\n\tСтоп! Непорядок со служебными файлами.\n" >&2
        exit 1
    fi
}

    # проверка наличия служебных файлов на диске
fn_check_service_files

    # импорт содержимого служебных файлов в этот скрипт
. $file_with_variables
. $file_with_functions

# - - - -

    # указывать каталоги без закрывающих слэшей

docs="/home/astenix/Документы"
docs_nas="$nas_host_tower/Документы"


file_list="aac,apk,arj,avi,bmp,bz2,dmg,flac,flv,gif,gz,heic,ico,iso,jar,jpeg,jpg,m4a,mkv,mov,mp3,mp4,mpeg,mpg,mts,ogg,pdf,png,rar,swf,tar,webm,webp,wma,wmv,xz,zip,zst,7z"

    # Скрипт запускается по алиасу в .bashrc, то есть, из неопределенного
    # каталога, поэтому сперва надо вычислить абсолютный путь к скрипту
    # (/home/mySettings/myScripts/nas/nas-backup-rsync.sh — задан через
    # алиас в ~/.bashrc) и уже от него строить путь к дочерним файлам вроде
    # exclude для LaTeX — отовсюду
script_dir="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"

    # 1.1 - ПЕРЕНЕСТИ файлы с ноутбука на NAS/ecnrypted
    # -------------------------------------------- #

    echo -e "\n\n\t${clr_AMBER}Перенести файлы с ноутбука на${clr_NONE} ${clr_RED_BRIGHT}NAS/ecnrypted${clr_NONE}"


        # --- Перенести содержимое /Документы в NAS/ecnrypted/Документы
        echo -e "\n\n\t${clr_GREEN_BRIGHT} ) Документы${clr_NONE}\nВсё перенести, несколько каталогов оставить.\n"

            # список исключаемых каталогов
            # без слэшей!
        docs_excluded_list=(
            "Mind Maps"
            "Автомобиль/_CarteaVerde"
        )

            # проверить наличие исключаемых каталогов;
            # полный стоп при любой ошибке
        fn_check_excluded_folders "$docs" "${docs_excluded_list[@]}"

            # создать массив исключаемых каталогов из
            # пользовательского списка через
            # вызов функции «fn_list_excluded_folders»
                # будет прочитана как поток
            # и передачи ей двух параметров:
                # • корневой каталог синхронизации
                # • список исключаемых каталогов
        readarray -t docs_excluded_array < <(fn_list_excluded_folders \
            "$docs" \
            "${docs_excluded_list[@]}")

        # перенести файлы из всех каталогов, кроме исключенных.
        # Остальные каталоги останутся пустыми, --remove-source-files
        # не трогает каталоги
        rsync -auzv \
            -e "ssh -i $ssh_key" \
            "${docs_excluded_array[@]}" \
            --skip-compress=$file_list \
            --remove-source-files \
            "$docs/" "$docs_nas/" | addTabs

        # 	скопировать оставшиеся файлы
        rsync -auzv \
            -e "ssh -i $ssh_key" \
            --skip-compress=$file_list \
            "$docs/" "$docs_nas/" | addTabs

            # удалить все пустые каталоги из корневого, кроме «исключённых» — в них остались файлы, поэтому они не попадают в этот фильтр
            # каталоги будут прочитаны «снизу вверх»
        find -L "$docs" -depth -type d -empty -delete # && echo -e "\nПустые каталоги удалены\n"
        
}}}
