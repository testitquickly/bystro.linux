====== Поиск файлов в KRunner ======

[ @k_runner @window @baloo @search ]

Этот поиск если и нужен, то его сперва надо очень аккуратно продумать. Оно в принципе полезно, когда в индексируемых каталогах только пара файлов. Когда их будет под тысячу, служба индексирования будет периодически отжирать по 10 Gb памяти под свои нужды без предупреждения, заставляя почти всё содержимое RAM перекатываться в swap.

Файл с индексом тоже потребует для себя места на диске. Когда файлов много, он вполне может разрастить до нескольких гигабайт.

==== Проверить состояние baloo ====

''balooctl status''

Если служба индексирования отключена, будет предложено её включить командой «balooctl enable». Но спешить не стоит.

==== Настройка индексирования ====

Если всё-таки хочется делать поиск файлов через KRunner:

''Параметры системы'' 
''> Рабочая среда''
''> Поиск''
''> Поиск файлов''
''> "Включить службу поиска файлов"''

Ещё здесь надо решить однозначно, нужно ли индексировать только файлы, или также их содержимое — для этого есть отдельная подгалочка. Второе в разы увеличивает нагрузку службы индексирования на систему, но не факт, что будет нужно в бытовых условиях.

=== Указать каталоги ===

в которых надо индексировать файлы, а также каталоги, в которых файлы индексировать НЕ надо. 

=== Определить зоны поиска ===

''Параметры системы''
''> Рабочая среда''
''> Поиск'' 
''> Поиск в Plasma''

Может быть разумно оставить следующие галочки:

* Документы
* Закладки
* Запуск программ
* Папки
* Последние Документы
* Приложения
* Расположения
* Точки входа
