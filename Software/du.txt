====== du ======

[ @debian @folder @du @baobab ] 

Disk Usage.

Есть более удобный для восприятия аналог: ''ncdu'' (см. далее).

В KDE есть гуи-аналог с названием „Disk Usage Analyzer”, он же ''baobab''. Зрительно поражает, на деле проще смотреть в консоль с ''du''.

===== du =====

==== Узнать размер текущего каталога (в kb) ====

''du''

=== Узнать размер каталога в более удобном формате (1K, 234M, 2G) ===

''du -h /home/user/folder''

=== Узнать „Total disk usage” ===

Иногда есть разница между размером файла\каталога и количеством занятого места на диске. 

''du -hc /home/user/folder''

=== Узнать размер каталога без перечисления всех его подкаталогов ===

''du -d 0 -h /home/user/folder''

На самом деле и тут происходит проверка всех дочерних каталогов и файлов, и если их общий размер огромен, то процесс будет длиться неопределенно долго.

===== ncdu =====

Псевдографический аналог du, более удобен для зрительного восприятия.

''sudo apt install ncdu''

Посмотреть размеры каталогов в корне (системный диск):

''sudo ncdu /''

Будет показано дерево каталогов и их размеры.

Справка:
* ?

Навигация стрелками:
* d — удалить каталог под курсором, 
* q — выйти.

Этот ''ncdu'' можно просто запустить в текущем каталоге, без параметров. 

Можно передвигаться по всем доступным подкаталогам, но текущий будет считаться корневым, подняться на уровень выше невозможно (выйти, перейти куда надо и там запустить; или запустить ''ncdu'' с указанием нужного каталога).
