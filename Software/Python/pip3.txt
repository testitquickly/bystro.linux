====== pip3 ======

[ @python @pip @pipx ]

	*todo Python pip — пересмотреть полностью

Это менеджер управления пакетами для Python. Язык развивается, и менеджер пакетов вместе с ним. ''pip3'' это менеджер пакетов для Python3.

===== Установить =====

''sudo apt install python3-pip''

Проверить версию (заодно узнать, где именно он живет):

''pip3 --version''

Ответ вроде ''pip 25.1.1 from /usr/lib/python3/dist-packages/pip (python 3.13)'' полностью годен.

===== Использовать =====

Через pip3 можно установить [[Виртуальные окружения]] Python и многое другое.

Но есть нюанс. Установить, например, пакет feedparser:

''pip3 install feedparser''

ВНЕЗАПНО нельзя, при попытке установить пакет будет предупреждение «''error: externally-managed-environment''». 

Внутри Debian много чего работает на Python (местами даже на Python2), и установка дополнительных пакетов прямо в систему может непредсказуемо повлиять на уже установленный софт, который использует Python. Поэтому все скрипты на Python, которым нужны какие-то пакеты, которые расширяют функциональность, в принципе надо запускать только внутри виртуального окружения — ''venv''. Все пакеты Python, которые могут понадобиться для выполнения скриптов, надо устанавливать в соответствующее виртуальное окружение каждого проекта по-отдельности, чтобы все нужные зависимости были в порядке (в изолированном окружении) и не влияли на Debian.

==== Установить пакеты Python глобально ====

Идея держать проекты Python в отдельных виртуальных окружениях масштабна и полезна, особенно если надо передать проект на другой компьютер. Но есть возможность установить какие-то пакеты в свою систему глобально, например [[Документирование кода:Zensical]] — ПО для генерирования сайта с документацией, возможности которого уместны не только для проектов Python, но и для проектов [[LaTeX]].

Использовать pipx — не преступление, но должно быть полностью на своём усмотрении.

Для этого нужен пакет ''pipx'':

''sudo apt install pipx''

Установить нужный пакет:

''pipx install --python /usr/bin/python3 zensical''

Всё будет установлено в каталог ''~/.local/share/pipx/venvs/zensical/''

Невозможно устанавливать плагины в софт, который установлен через pipx, их туда надо «впихнуть» (to inject). 

''source ~/.local/share/pipx/venvs/zensical/bin/activate''

и потом добавлять нужное

''pipx inject zensical mkdocstrings[python] mkdocs-material mkdocs-awesome-pages-plugin mkdocs-add-number-plugin''

==== Посмотреть список установленного ПО в окружение pipx ====

Надо сперва активировать соответствующее виртуальное окружение. Например, надо проверить установленный софт, которые касается, например, только пакета MkDocs:

''source ~/.local/share/pipx/venvs/mkdocs/bin/activate''

Посмотреть:

''python -m pip list | grep mkdocs''

==== Удалить отдельный плагин ====

…например ''mkdocs-awesome-nav'' — это надо сделать в том же виртуальном окружении:

''pipx uninject mkdocs mkdocs-awesome-nav''
