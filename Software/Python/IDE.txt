====== IDE ======

[ @python @development @ide ]

Интегрированная среда разработки — IDE.

* **VS Code**

Она же «Visual Studio Code», легковесная IDE от Microsoft, не путать со здоровенной «Visual Studio». Free and built on open source. Integrated Git, ИИ (по-умолчанию github “Copilot”, даже сам). Очень много плагинов. И оооочень много интеграции с внешними ресурсами. В базе работает незаметно-шустро, но если перевесить плагинами, то будет тяжко — работает на Electron.

* **PyCharm**

IDE от JetBrains. На Java. Есть PyCharm Community (бесплатное) — venv, БД, линтеры и форматтеры (pylint, mypy, black, ruff и др.), Jupyter Notebook; и PyCharm Professional (платное, подписка) — то же, + поддержка фреймворков (Django, Flask) и AI. Мощно, дорого, жрет ресурсы.

* [[KDevelop]]

Нативное, и поэтому шустрое KDE-software для программистов на C; окружение для Python доступно через плагин — ''sudo apt install kdevelop kdevelop-python'' — при создании нового проекта надо сперва выбрать тип проекта ’Python’. Встроенный линтер только один, ''pycodestyle'', и его можно или включить (этого, в принципе, достаточно), или выключить, все остальные линтеры можно использовать только через консоль как внешние утилиты. И вообще почти все инструменты для разработки в KDevelop недоступны, только консоль (лучше внешняя, внутренняя живет отдельной жизнью). Старик стабильно тащщит, но его последний релиз был в 2020-м году. Крэши — регулярно.

* **Eric**

Полностью на Python, есть встроенные инструменты для просмотра статистики кода, для анализа кода, встроенный генератор документации по проекту в html… Выглядит outdated, стремительно устаревает.

* **Eclipse**

Древняя тема, на Java, но работает. Ресурсожрателен. Сегодня смотрится странно, а когда-то был стандартом.

* [[Sublime]]

Текстовый редактор. Без плагинов — тупо текстовый редактор, с плагинами — шатл «Буран». Шустр.

* **Kate**

Стандартный текстовый редактор в KDE превращается в IDE после установки плагина ''kate-lsp'' (Language Server Protocol Client), которое приносит автодополнение, линтинг, переходы, подсказки и другие IDE-функции. Шустр, бо нативен. 

* **vim**

Текстовый, кагбэ, редактор. Всегда готов ассимилировать ценителей непереходящих ценностей. Resistance is futile.
