Вопрос привычек, удобства и требуемых возможностей. 

Вообще будет работать почти всё, что на глаза попадётся:
* “Eclipse” — древний, но работает
* “Eric” (есть и в репозитории Debian) — полностью написана на Python, есть множество инструментов для просмотра статистики кода, для анализа кода, для генерирования документации по проекту в html…
* “pyCharm” — 
* “[[Sublime]]”
* нативное [[KDevelop]] (окружение для Python доступно через плагин: ''sudo apt install kdevelop kdevelop-python'' и при создании нового проекта надо сперва выбрать тип проекта ’Python’


Например, после [[Kile]] в KDevelop всё удобно и понятно, но если нужны какие-то особенные средства проверки кода «на лету», то может быть сложно. Там есть встроенный аналайзер, который можно только включить/выключить — основан на /usr/bin/pycodestyle и работает как стандартный pep8, и работает, даже если pep8 в системе не установлен.
