====== IDE ======

[ @python @development @ide ]

Интегрированная среда разработки — IDE.

Вопрос привычек, удобства и требуемых возможностей. 

Вообще будет работать почти всё, что на глаза попадётся:
* Eclipse — древняя, на Java, но работает.
* Eric — есть в репозитории Debian, полностью написана на Python; есть встроенные инструменты для просмотра статистики кода, для анализа кода, встроенный генератор документации по проекту в html…
* Visual Studio Code
* PyCharm — IDE от JetBrains на Java. Есть PyCharm Community (бесплатное) — venv, БД, линтеры и форматтеры (pylint, mypy, black, ruff и др.), Jupyter Notebook; и PyCharm Professional (платное, подписка) — то же, + поддержка фреймворков (Django, Flask) и AI.
* [[Sublime]] — блокнот с возможностью перестроить его как полноценное IDE
* [[KDevelop]] — нативное, и поэтому шустрое KDE-software для программистов на C; окружение для Python доступно через плагин — ''sudo apt install kdevelop kdevelop-python'' — при создании нового проекта надо сперва выбрать тип проекта ’Python’. Встроенный линтер ''pycodestyle'', и его можно или включить (этого, в принципе, достаточно), или выключить, все остальные линтеры доступны только через консоль как внешние утилиты. И вообще почти все инструменты для разработки с KDevelop остаются внешними. Но можно подключить AI.
* vim
