====== Micro ======

[ @micro ] 

Текстовый редактор. Не идеальный, но очень близок к этому. Очень небольшой, кроссплатформенный. 

Киллер-фич сразу несколько — можно передвигать вверх-вниз строки (Alt+Up/Down или назначить свой), большинство хоткеев максимально привычные (Ctrl+S, Ctrl+C, Ctrl+V). Шрифт наследуется из настроек профиля [[Konsole]] (или в чем будет открыт терминал). Размер шрифта меняется по Ctrl+/-. 

Управляется хоткеями, наследует подход nanо/vim в переключении в режим управления, в одном окне открывает панели, есть плагины и можно тыкать мышкой, поэтому удобно пролистывать тексты или перемещать фокус на нужную панельку в управлении с тачпада или мышью.

Чего нет — управления файлами на диске и россыпи плагинов для управления спутниками. Это текстовый редактор, а не emacs.

Использовать Micro вместе с [[Midnight Commander]] надо очень аккуратно — при конфликте хоткеев MC перебьёт Micro. Например, Ctrl+O в MC свёртывает/разворачивает манагер, а в Micro Ctrl+O нужен для открытия файлов. После появления/убирания MC Mirco всё еще работает, но восстановить его целиком будет сложновато.

===== Установить =====

С https://github.com/micro-editor/micro/releases снять последний релиз, который подходит для системы (например, micro-2.0.15-linux64.tar.gz) и распаковать в ''/opt'' 

В ''$HOME/.bashrc'' добавить алиас 

''alias m='/opt/micro/micro' # старт text editor “Micro”''

Для доступа к системному буферу обмена под X11:

''sudo apt install xclip''

Под Wayland: установить ''wl-clipboard''

Теперь просто “''m''” откроет пустой редактор Micro. А ”''m file.txt''” откроет file.txt в окне Micro. 

===== Объявить Micro главным текстовым редактором в системе =====

См. настройку главного текстового редактора в [[Midnight Commander#Объявить mcedit основным редактором]] — надо повторить процедуру, только вместо ''mcedit'' надо объявить ''/otp/micro/micro''

Внимание к тому, что если Micro будет назначен основным редактором в терминале (в системе), то открывать его MC можно спокойно, все внутренние команды вроде Ctrl+O будут работать. 

Но если
 
1. открыть MC
2. свернуть его по Ctrl+O
3. отдельно запустить Micro (''m abc.txt'')

то сочетание Ctrl+O в Micro (Открыть файл) будет перехвачено MC, который получит команду «развернуться обратно», а Micro в принципе останется открытым, и файл в нем всё еще будет открытым на редактирование, но вернуть Micro на экран будет затруднительно.

===== Подтянуть свою цветовую схему =====

Напрямую перенести какую-то схему из Kate (Oblivion) в Micro не получится, потому что Kate использует десятки семантических типов подсветки, а micro — гораздо меньше. Можно сделать компромиссный вариант:

''mkdir ~/.config/micro/colorschemes/ && m ~/.config/micro/colorschemes/oblivion.micro''

Вставить это:

''color-link default "#d3d7c1,#201f1f"''
''color-link comment "#30a100,#201f1f"''
''color-link identifier "#d3d7c1,#201f1f"''
''color-link statement "bold #ffffff,#201f1f"''
''color-link symbol "#eeeeec,#201f1f"''
''color-link preproc "#ad7fa8,#201f1f"''
''color-link type "#508ed8,#201f1f"''
''color-link special "#729fcf,#201f1f"''
''color-link constant "bold #edd400,#201f1f"''
''color-link string "#edd400,#201f1f"''
''color-link char "#ce5c00,#201f1f"''
''color-link number "#edd400,#201f1f"''
''color-link statusline "#ffffff,#2e3436"''
''color-link tabbar "#e0dedb,#302f2f"''
''color-link line-number "#e0dedb,#201f1f"''
''color-link current-line-number "#ffffff,#201f1f"''
''color-link cursor-line "#201f1f,#2e3436"''
''color-link selection "#eeeeec,#184880"''
''color-link error "#e85848,#201f1f"''

Два цвета в каждой строке это текст (foreground) и фон (background). Можно подобрать подсветку строки, в которой находится курсор через 
''color-link cursor-line "…,…"''

Затем выбрать эту схему:

''> set colorscheme oblivion''

===== Управление =====

==== Открыть строку ввода команд редактору ====

''Ctrl-e'' и набрать нужную команду (например ''help'').

Посмотреть список доступных команд — [Tab].

В прицнипе всё автодополнение по [Tab] — или команд, или доступных опций для какой-то команды, или перечисление каталогов/файлов при поиске нужного.

==== Пролистать уже введенные команды ====

''Ctrl-e >'' клавиша [Up]

Сохранить: ''Ctrl-s'' 

Поменять цветовую схему: ''Ctrl+e > set colorscheme''

==== Открыть строку ввода команд в консоли ====

''Ctrl+b''

Ввести что надо (ls -a), получить результат, вернуться в Micro по [Enter].

==== Открыть файл ====

''Ctrl+e > filename.txt''

или 

''Ctrl+o'' 

Учесть, что если в консоли уже был открыт [[Midnight Commander]], свернут и после него открыт Micro, то на ''Ctrl+o'' отреагирует MC (развернется), и окно редактора просто закроется, изменения будут потеряны.

Можно начать набирать первые символы, затем [Tab]

==== Открыть панели ====

* вертикально: ''Ctrl+e > vsplit''
* горизонтально: ''Ctrl+e > hsplit''

Автокомплит по [Tab] срабатывает после ''> vs…''

Переключиться между панелями: ''Ctrl-w''

Осторожно, ''Ctrl-q'' закрывает панели без подтверждения, если в файле не было изменений.

Открыть в другой панели какой-то файл: 

''Ctrl+e > vsplit filename.txt''

или назначить свои хоткеи на открытие панелей.

==== Вкладки ====

Работает только на англоязычной раскладке.

Передвигать или переименовывать вкладки невозможно. Вкладки фиксируются в порядке открытия.

Ширина вкладки занимет полное название открытого в ней файла. Если файл называется длинно — его вкладка будет широченной.

* Открыть новую: ''Ctrl-t''
* Переключиться по вкладкам вправо: ''Alt-,''
* Влево: ''Alt-.''

Удобнее перебиндить переключение между ними на 

   ''"Ctrl-Left": "PreviousTab",''
   ''"Ctrl-Right": "NextTab"''

Что любопытно: по ''Alt-.'' переключение между вкладками циклично-бесконечное. Если назначить свои клавиши (любые!) и нажимать ''"NextTab"'', то бесконечного цикла не будет — переклюк дойдет до последней вкладки в ряду и остановится.

==== Сессия ====

Концептуально отсутствует, но можно открыть в Micro перечень файлов

''m file1.txt file2.txt file3.txt''

они откроются во вкладках — вот и сессия. 

Можно открыть все файлы в каталоге:

''m *''

Вкладки невозможно перемещать, можно только сразу задать порядок их открытия. Здесь первая вкладка будет занята файлом “''gitara.txt''”, за ним подтянутся остальные в алфавитной сортировке (как файлы в MC). Конечно, файл “''gitara.txt''” будет открыт и второй раз:

''m gitara.txt *''

Если файлы находятся в разных местах, то их удобнее закатать в sh-скрипт:

''#!/bin/bash''
''m \''
  ''~/notes/todo.txt \''
  ''~/notes/ideas.txt \''
  ''~/project/README.md''

Для иллюзорного удобства «имени вкладки» можно убрать расширение из имени файл, редактор открывает все файлы в plain text.

==== Подсветка текста ====

В ''~/.config/micro/settings.json'' добавить приказ воспринимать синтаксис любого файла как shell:

''{''
	''"colorscheme": "oblivion",''
	''"filetype": "shell"''
''}''

Теперь текст любого файла, невзирая на его расширение, будет подсвечен как скрипт ''.sh''

Если к какому-то файлу надо применить другую подсветку текста, например, ''yaml'':

''Ctrl+E''
''set filetype yaml''

Все файлы с синтаксисом см. https://github.com/micro-editor/micro/blob/master/runtime/syntax/README.md

==== Строки ====

* Передвинуть строку вверх: ''Alt-[Up]''
* Вниз: ''Alt-[Down]''

Можно переназначить, как удобнее.

==== Нумерация строк ====

Вкл/Выкл:

''Ctrl+R''

==== Поиск текста ====

''Ctrl+F''

В строке ввода (внизу) набрать поисковый запрос.

==== Замена текста ====

Открыть поиск:

''Ctrl+F''

Переключиться на замену:

''Ctrl+R''

В строке ввода (внизу):

* ввести то, что надо заменить
* [Tab]
* ввести то, на что надо заменить
* подтверждение замены: ''Enter''

Для замены всех вхождений:

''Alt+A''

(или Esc, затем A — зависит от терминала)

Можно сделать автозамену по тексту из командной строки (работает не во всех сборках micro одинаково; похоже на sed, но не sed):

''micro file.txt +':%s/old/new/g'''

==== Функциональные клавиши ====

Работают не во всех терминалах.

F1 	Open help
F2 	Save
F3 	Find
F4 	Quit
F7 	Find
F10 	Quit

==== Выход ====

''Ctrl-q''

===== Плагины =====

Посмотреть доступные с кратким описанием:

''Ctrl+E > plugin search''

Самые значимые:

* manipulator

* snippets

* wc (word count)

Разочарование:

* **filemanager** — хэлпа нет; вызывается по команде ''tree''; возможен только просмотра файлов в текущем каталоге; переключения нет; открывать файлы надо через ''Ctrl+o''

* **bookmark** — перехватывает сочетания клавиш, надо настраивать

Установить нужный:

''Ctrl+E > plugin install manipulator filemanager bookmark snippets wc'' 

Посмотреть возможности определенного плагина:

''Ctrl+E > plugin manipulator''

===== Горячие клавиши =====

''mcedit ~/.config/micro/bindings.json''

Это строгий JSON, комментарии (// или /* */) не поддерживаются, любые строки кроме JSON вызовут ошибку загрузки биндингов. После последнего вхождения запятой быть не должно.

Пример — отменить дефолтные хоткеи передвижения строк через Alt+Up/Down и назначить это всё на Ctrl+Up/Down:

''{''
	''"Alt-Up": "None",''
	''"Alt-Down": "None",''
   ''"Ctrl-Up": "MoveLinesUp",''
	''"Ctrl-Down": "MoveLinesDown",''
	''"Ctrl-Insert": "DeleteLine",''
	''"Ctrl-Left": "PreviousTab",''
	''"Ctrl-Right": "NextTab"''
''}''

Можно назначить команду:

''{''
	''"Alt-p": "command:pwd"''
''}''

Можно назначить цепочку команд:

''{''
	''"Alt-s": "Save,Quit"''
''}''

Есть большой [[https://github.com/micro-editor/micro/blob/master/runtime/help/keybindings.md#bindable-actions-and-bindable-keys|список]] команд, которые можно связать с горячими клавишами. Там же список клавиш и сочетаний клавиш, на которые можно навесить команды. Аккуратно соотнести команды/клавиши в редакторе с клавишами в [[tmux]] или [[Midnight Commander]], чтобы не перекрыли друг друга.

 После редактирования файла с кейбиндингами Micro надо перезапустить полностью.
