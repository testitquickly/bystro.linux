====== Очистить кэш Firefox ======

[ @firefox @settings ]

===== Из браузера =====

Стандартный способ.

1. Открыть about:preferences#privacy
2. найт раздел «Куки и данные сайтов» и подождать, когда закончится вычисление объёма кэша и данных сайтов
3. нажать [Удалить данные]
4. отжать галочу для куки, оставить галочку для кэша
5. [Удалить]

===== Через расширение «Clear Cache» =====

https://addons.mozilla.org/en-US/firefox/addon/clearcache/?ref=itsfoss.com

При нажатии на иконку (или на F9) происходит моментальное очищение кэша, без предупреждения и без подтверждения выполненной задачи. 

Куки это расширение не трогает.

===== Из консоли =====

Это выполняется при незапущенном браузере. 

Сперва надо точно знать каталог, в котором находится профиль — открыть страницу ''about:profiles.'' 

Если установка стандартная, то «корневой каталог» будет называться вроде ’''rvhe3ml0.Default User''’ (обратить внимание на пробел в названии), а «Локальный каталог», в котором находится кэш бразуера, будет находиться в каталоге „''~/.cache/mozilla/firefox/rvhe3ml0.Default User/”''

''rm -rf ~/.cache/mozilla/firefox/rvhe3ml0.Default\ User/cache2/*''

Можно прицелиться на нужный каталог через слово ’Default’:

''rm -rf ~/.cache/mozilla/firefox/*.Default*/cache2/*''

Если установка нестандартная, то смотрим сразу в каталог профиля. В каталоге профиля находится подкаталог ‘cache2‘, и нам надо удалить его содержимое (выполнять только из-под своего юзера).

==== Удалить куки ====

''rm ~/.mozilla/firefox/rvhe3ml0.Default\ User/cookies.sqlite''

===== Проверить журнал запуска =====

Если браузер запускается ДОЛГО, в первую очередь надо посмотреть в журнал действий:

''Инструменты'' 
''> Инструменты браузера''
''> Консоль бразуера''
''> Мультипроцессный''

Откроется много всякого интересного, и будет очевидно, на каких расширениях происходит замедление.
